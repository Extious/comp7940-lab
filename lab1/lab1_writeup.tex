\section*{3. Why SSH Keys Are More Secure Than Passwords}
\begin{itemize}
  \item SSH uses public-key (asymmetric) cryptography: only the public key is uploaded; the private key stays on your computer.
  \item The private key is never sent over the network, which reduces the risk of credential interception.
  \item SSH keys can be protected with a local passphrase; even if the key file is copied, it is harder to use without the passphrase.
  \item Keys are resistant to password reuse and common phishing attacks because there is no typed password to steal.
  \item Keys can be revoked/rotated per device without changing other devices' access.
\end{itemize}
